🔐 认证说明 Authentication

系统采用 Token 认证机制。用户登录成功后获取 Token，后续请求需携带 Token 进行身份验证。Token 可通过以下三种方式传递：

HTTP Header（推荐）

在请求头中添加 Token 字段：

token: {your_token_here}

Query Parameter

在 URL 查询参数中传递：

/api/user/index?token={your_token}

Cookie

在 Cookie 中设置 Token：

Cookie: token={your_token}

Token 有效期

Token 默认有效期为 7 天，可通过 /api/token/refresh 刷新延长至 30 天。

📦 响应格式 Response Format

所有接口统一返回 JSON 格式数据，包含状态码、消息、时间戳和数据。

1
成功 Success
0
失败 Failure
401
未授权 Unauthorized
403
已禁止 Forbidden
JSON Response Structure
{
    "code": 1,
    "msg": "success",
    "time": 1720000000,
    "data": {
        // 业务数据
    }
}
