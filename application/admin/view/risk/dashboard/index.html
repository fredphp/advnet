风控系统工作流程

1
用户行为采集

实时采集用户行为数据，包括视频观看、任务完成、提现申请、红包领取、邀请行为等，记录行为时间、频次、设备信息等。

2
规则引擎检测

根据预设的风控规则，对用户行为进行实时检测。规则按类型分为视频、任务、提现、红包、邀请、全局六大类。

3
风险评分计算

触发规则后，根据规则的权重分值累加用户风险分。支持分数衰减机制，长期无违规可逐步降低风险分。

4
自动处罚执行

根据风险分阈值自动触发处罚：警告、拦截、冻结、封禁。处罚时长根据违规程度和次数动态计算。

风险评分规则

风险分范围 风险等级 系统响应
0 - 49 安全 正常使用，无任何限制
50 - 149 低风险 记录日志，暂不处罚
150 - 299 中风险 重点监控，触发警告
300 - 499 高风险 自动冻结账户，限制提现
500+ 危险 自动封禁账户，加入黑名单
分数衰减机制：用户连续7天无违规行为，风险分自动降低10%；连续30天无违规，风险分降低50%。

处罚动作说明

警告 (warn)

仅记录违规日志，不限制用户操作。用于低风险行为的提醒。

拦截 (block)

拦截当前操作请求，禁止该行为执行。如禁止本次提现、禁止本次任务提交等。

冻结 (freeze)

暂时冻结账户功能，期间无法提现、无法完成任务。可设置冻结时长，到期自动解冻。

封禁 (ban)

永久或长期禁止用户使用平台。封禁用户自动加入黑名单，清除登录Token，冻结资产。
