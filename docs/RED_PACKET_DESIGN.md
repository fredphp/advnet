# 红包任务系统设计文档

## 一、数据结构

### 1. 核心表结构

#### 1.1 红包任务表 (advn_red_packet_task)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 任务名称 |
| task_type | VARCHAR(30) | 任务类型: download_app/mini_program/play_game/watch_video/share_link/sign_in |
| task_url | VARCHAR(500) | 任务跳转链接 |
| task_params | TEXT | 任务参数(JSON) |
| total_amount | DECIMAL(18,2) | 红包总金额(金币) |
| total_count | INT | 红包总数量 |
| single_amount | DECIMAL(18,2) | 单个红包金额 |
| required_duration | INT | 要求时长(秒) |
| verify_method | VARCHAR(30) | 验证方式: auto/manual/third_party |
| audit_type | VARCHAR(20) | 审核方式: auto/manual |
| user_limit | INT | 每人可领取次数 |
| start_time | INT | 开始时间 |
| end_time | INT | 结束时间 |
| status | TINYINT | 状态: 0=禁用, 1=进行中, 2=已结束, 3=已抢完 |

#### 1.2 任务参与记录表 (advn_task_participation)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| task_id | INT | 任务ID |
| user_id | INT | 用户ID |
| order_no | VARCHAR(32) | 订单号 |
| status | TINYINT | 状态: 0=已领取待完成, 1=已完成待审核, 2=审核通过待发放, 3=已发放, 4=审核拒绝, 5=已过期, 6=已取消 |
| start_time | INT | 开始时间 |
| end_time | INT | 完成时间 |
| duration | INT | 实际耗时(秒) |
| reward_coin | DECIMAL(18,2) | 奖励金币 |
| screenshot_urls | TEXT | 截图URLs(JSON) |
| proof_data | TEXT | 证明数据(JSON) |

## 三、API接口设计

### 1. 用户端API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/redpacket/tasks` | GET | 获取任务列表 |
| `/api/redpacket/detail` | GET | 获取任务详情 |
| `/api/redpacket/categories` | GET | 获取任务分类 |
| `/api/redpacket/receive` | POST | 领取任务 |
| `/api/redpacket/submit` | POST | 提交任务完成 |
| `/api/redpacket/records` | GET | 获取参与记录 |
| `/api/redpacket/today` | GET | 获取今日统计 |
| `/api/redpacket/cancel` | POST | 取消任务 |

## 四、防作弊方案

### 1. 多维度检测机制

```
┌────────────────────────────────────────────────────────────────────────┐
│                          防作弊检测体系                                │
└────────────────────────────────────────────────────────────────────────┘

                                  用户请求
                                     │
                                     ▼
        ┌────────────────────────────────────────────────────────┐
        │                 第一层: 网关检测                       │
        │  • IP黑名单/地理位置/代理检测                          │
        │  • 设备指纹/多开检测/模拟器检测                         │
        │  • 请求频率/并发限制                                   │
        │  • 签名验证/时间戳校验                                  │
        └────────────────────────────────────────────────────────┘
                                     │
                                     ▼
        ┌────────────────────────────────────────────────────────┐
        │                 第二层: 业务检测                       │
        │  • 账户年龄/实名认证/历史记录                           │
        │  • 领取次数/任务限制/时间限制                           │
        │  • 操作时间/行为轨迹/异常行为                           │
        │  • 邀请关系/设备关联/IP关联                             │
        └────────────────────────────────────────────────────────┘
                                     │
                                     ▼
        ┌────────────────────────────────────────────────────────┐
        │                 第三层: 任务验证                       │
        │  • 时长验证(播放/游戏/下载)                             │
        │  • 截图验证(OCR识别/图片对比)                           │
        │  • 回调验证(第三方/服务端)                              │
        │  • 证明验证(包名/进程检测)                              │
        └────────────────────────────────────────────────────────┘
                                     │
                                     ▼
        ┌────────────────────────────────────────────────────────┐
        │                 第四层: 风险评估                       │
        │                                                        │
        │   风险分 = Σ (因子权重 × 因子得分)                     │
        │   • 设备风险因子 (0-25分)                               │
        │   • 账户风险因子 (0-25分)                               │
        │   • 行为风险因子 (0-25分)                               │
        │   • 关联风险因子 (0-25分)                               │
        │                                                        │
        │   阈值: >= 70分拦截, 40-70分人工审核, <40分自动通过    │
        └────────────────────────────────────────────────────────┘
```

### 2. 具体防作弊规则

| 类型 | 规则 | 默认值 |
|------|------|--------|
| IP限制 | 同IP每日领取限制 | 50次 |
| IP限制 | 同IP关联账户限制 | 10个 |
| 设备限制 | 同设备每日领取限制 | 20次 |
| 设备限制 | 同设备关联账户限制 | 3个 |
| 时间检测 | 任务最短完成时间 | 10秒 |
| 时间检测 | 异常速度阈值 | 0.3 |

### 3. 任务验证规则

**下载App任务:**
- 检查应用是否已安装(包名验证)
- 记录首次安装时间
- 验证应用启动时长
- 设备进程检测

**小程序任务:**
- 小程序跳转回调验证
- 验证小程序AppID
- 记录停留时长
- 小程序服务端通知

**游戏任务:**
- 游戏SDK时长统计
- 心跳包验证(每30秒)
- 游戏行为日志
- 防修改时间检测

### 4. 异常处理

```
风险评分 >= 70:  拦截领取 + 记录日志 + 加入观察名单
风险评分 40-70:  允许领取 + 人工审核 + 增加验证难度
风险评分 < 40:   正常流程 + 自动审核
```
